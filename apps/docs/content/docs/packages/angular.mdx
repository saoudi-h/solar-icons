The `@solar-icons/angular` package provides Angular components for Solar Icons. It's designed for Angular 17+ with modern patterns including signals, standalone components, and optimal performance.

## Installation

```package-install
@solar-icons/angular
```

## Features

- **Modern Angular 17+** → Uses signals (`input()`), standalone components, and `hostDirectives`
- **Minimal Bundle** → Tree-shakeable, only imported icons are included
- **Accessibility** → Built-in title support for screen readers
- **TypeScript** → Full type safety with IntelliSense support

## Basic Usage

Import icon components and use them in your templates:

```tsx
import { Component } from '@angular/core';
import { HeartBold } from '@solar-icons/angular';

@Component({
  selector: 'app-root',
  standalone: true,
  imports: [HeartBold],
  template: `
    <svg solarHeartBold [size]="24" [color]="'#ef4444'" />
  `,
})
export class AppComponent {}
```

## Import Patterns

All icons use **explicit global names** that include the style suffix (e.g., `HeartBold`, `ArrowLeftLinear`). This maps directly to the attribute selector you use in templates.

**From the root (any icon)**
```tsx
import { HeartBold, HeartLinear, HeartOutline } from '@solar-icons/angular';
import { StarBold, StarLinear } from '@solar-icons/angular';
```

**By category**
```tsx
import { HeartBold, HeartLinear } from '@solar-icons/angular/like';
import { ArrowLeftBold, ArrowRightLinear } from '@solar-icons/angular/arrows';
```

> **Note:** Per-style imports (`@solar-icons/angular/Bold`) are intentionally not provided. Angular icons are identified by their full global name in templates — importing `Heart` from `/Bold` would mislead you into writing `<svg solarHeart />` when the correct selector is `<svg solarHeartBold />`. Explicit names eliminate this ambiguity.

Each icon uses a camelCase attribute selector with the "solar" prefix:

```html
<svg solarHeartBold />
<svg solarAltArrowDownBold />
```

## Props

All icon components accept the following inputs:

| Input | Type | Default | Description |
|-------|------|---------|-------------|
| `size` | `string \| number` | `'1em'` | Icon size (e.g., `24`, `'2rem'`) |
| `color` | `string` | `'currentColor'` | Icon color (any valid CSS color) |
| `alt` | `string \| null` | `null` | Accessible label (adds `<title>` element) |
| `mirrored` | `boolean` | `false` | Flip icon horizontally |

### Examples

```tsx
// Size as number (pixels)
<svg solarHeartBold [size]="32" />

// Size as string
<svg solarHeartBold [size]="'2rem'" />

// Custom color
<svg solarHeartBold [color]="'#ef4444'" />

// Accessibility
<svg solarHeartBold [alt]="'Favorite'" />

// Mirrored
<svg solarArrowLeftBold [mirrored]="true" />

// Combined
<svg solarHeartBold 
  [size]="24" 
  [color]="'#ef4444'" 
  [alt]="'Add to favorites'" />
```

## Accessibility

For accessibility, use the `alt` input to provide a descriptive label:

```tsx
<svg solarHeartBold [alt]="'Add to favorites'" />
```

This will:
- Add a `<title>` element inside the SVG
- Remove `aria-hidden="true"` attribute
- Make the icon accessible to screen readers

Without `alt`, icons are automatically marked as decorative with `aria-hidden="true"`.

## Dynamic Icons

To render icons dynamically based on runtime strings (e.g., from a database), use the built-in `SolarDynamicIcon` directive.

### 1. Register Icons

First, provide the icons you want to make available for dynamic rendering in your component or app providers:

```tsx
import { provideSolarIcons } from '@solar-icons/angular';
import { HeartBold, StarBold, HomeBold } from '@solar-icons/angular/like';

bootstrapApplication(App, {
  providers: [
    provideSolarIcons({ HeartBold, StarBold, HomeBold })
  ]
});
```

### 2. Render the Icon

Use the `solarIcon` directive in your template. It accepts either an icon name (string) or an icon component class.

```tsx
import { Component } from '@angular/core';
import { SolarDynamicIcon } from '@solar-icons/angular';

@Component({
  selector: 'app-root',
  standalone: true,
  imports: [SolarDynamicIcon],
  template: `
    <!-- Render by registered name -->
    <ng-container solarIcon="HeartBold" [size]="32" color="#ef4444" />

    <!-- Render from a dynamic variable -->
    <ng-container [solarIcon]="dynamicIconName" [size]="24" />

    <!-- Render by component class directly -->
    <ng-container [solarIcon]="HeartBold" />
  `,
})
export class AppComponent {
  dynamicIconName = 'StarBold';
}
```

## Angular Version Compatibility

This package supports **Angular 17+** (tested with Angular 17, 18, 19, 20, and 21). It uses modern Angular features:

- **Signals** → `input()` instead of `@Input()`
- **Standalone Components** → No NgModule required
- **Host Directives** → Composition via `hostDirectives`
- **Effect** → Reactive title management

## Performance

Icons are optimized for performance:

- **Tree-shakeable** → Only imported icons are included in your bundle
- **Minimal overhead** → Simple directive for title management
- **Compiled templates** → SVG structure is part of the component template

## Package Size

The package includes ~7,476 icon components (1,246 icons × 6 styles). However, with tree-shaking, only the icons you import will be included in your final bundle.
