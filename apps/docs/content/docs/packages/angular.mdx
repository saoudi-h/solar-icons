The `@solar-icons/angular` package provides Angular components for Solar Icons. It's designed for Angular 17-21 with modern patterns including signals, standalone components, and optimal performance.

## Installation

```package-install
@solar-icons/angular
```

## Features

- **Modern Angular 17-21** → Uses signals (`input()`), standalone components, and `hostDirectives`
- **Minimal Bundle** → Tree-shakeable with compile-time templates (no runtime parsing)
- **Accessibility** → Built-in title support for screen readers
- **TypeScript** → Full type safety with IntelliSense support
- **Flexible Imports** → Import by style, category, or individual icons

## Basic Usage

Import icons from the package and use them as Angular components:

```tsx
import { Component } from '@angular/core';
import { HeartBold } from '@solar-icons/angular/Bold';

@Component({
  selector: 'app-root',
  standalone: true,
  imports: [HeartBold],
  template: `
    <svg solarHeartBold [size]="24" [color]="'#ef4444'" />
  `,
})
export class AppComponent {}
```

## Import Patterns

### By Style

Import all icons of a specific style:

```tsx
import { HeartBold, StarBold, UserBold } from '@solar-icons/angular/Bold';
import { HeartLinear, StarLinear } from '@solar-icons/angular/Linear';
```

### By Category

Import icons from a specific category:

```tsx
import { Bold } from '@solar-icons/angular/like';

// Use icons from the category
@Component({
  imports: [Bold.Heart, Bold.Star],
  template: `
    <svg solarHeartBold />
    <svg solarStarBold />
  `,
})
```

### Global Import

Import all icons with disambiguated names:

```tsx
import { HeartBold, HeartLinear, HeartOutline } from '@solar-icons/angular';
```

## Props

All icon components accept the following inputs:

| Input | Type | Default | Description |
|-------|------|---------|-------------|
| `size` | `string \| number` | `'1em'` | Icon size (e.g., `24`, `'2rem'`) |
| `color` | `string` | `'currentColor'` | Icon color (any valid CSS color) |
| `alt` | `string \| null` | `null` | Accessible label (adds `<title>` element) |
| `mirrored` | `boolean` | `false` | Flip icon horizontally |

### Examples

```tsx
// Size as number (pixels)
<svg solarHeartBold [size]="32" />

// Size as string
<svg solarHeartBold [size]="'2rem'" />

// Custom color
<svg solarHeartBold [color]="'#ef4444'" />

// Accessibility
<svg solarHeartBold [alt]="'Favorite'" />

// Mirrored
<svg solarArrowLeftBold [mirrored]="true" />

// Combined
<svg solarHeartBold 
  [size]="24" 
  [color]="'#ef4444'" 
  [alt]="'Add to favorites'" />
```

## Accessibility

For accessibility, use the `alt` input to provide a descriptive label:

```tsx
<svg solarHeartBold [alt]="'Add to favorites'" />
```

This will:
- Add a `<title>` element inside the SVG
- Remove `aria-hidden="true"` attribute
- Make the icon accessible to screen readers

Without `alt`, icons are automatically marked as decorative with `aria-hidden="true"`.

## Dynamic Icons

To load icons dynamically based on runtime data:

```tsx
import { Component, Type } from '@angular/core';
import * as solar from '@solar-icons/angular';

@Component({
  selector: 'app-dynamic-icon',
  standalone: true,
  template: `
    @if (iconComponent) {
      <svg [solarIcon]="iconComponent" [size]="size" [color]="color" />
    }
  `,
})
export class DynamicIconComponent {
  iconComponent: Type<unknown> | undefined;
  size = 24;
  color = 'currentColor';

  loadIcon(name: string, style: string) {
    const globalName = name + style; // e.g., "HeartBold"
    this.iconComponent = (solar as Record<string, unknown>)[globalName] as Type<unknown>;
  }
}
```

## Angular Version Compatibility

This package supports Angular 17.x through 21.x. It uses modern Angular features:

- **Signals** → `input()` instead of `@Input()`
- **Standalone Components** → No NgModule required
- **Host Directives** → Composition via `hostDirectives`
- **Effect** → Reactive title management

## Performance

Icons are optimized for performance:

- **Compile-time templates** → SVG structure is compiled by Angular Ivy, not created at runtime
- **Tree-shakeable** → Only imported icons are included in your bundle
- **No runtime parsing** → Icon data is pre-processed at build time
- **Minimal overhead** → Simple directive for title management

## Package Size

The package includes ~9,000 icon components (1,500 icons × 6 styles). However, with tree-shaking, only the icons you import will be included in your final bundle.

Example bundle impact:
- Single icon: ~1-2 KB
- 10 icons: ~10-15 KB
- 100 icons: ~100-120 KB
