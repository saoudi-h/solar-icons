Solar Icons v3 works in server-side rendering environments without configuration.

## React Server Components

```tsx
import { HeartIcon } from '@solar-icons/react/bold/heart'

export default function ServerComponent() {
    return <HeartIcon color="#ef4444" size={24} />
}
```

## Next.js

No special setup required. Icons render on the server and hydrate on the client.

```tsx
import { SettingsIcon } from '@solar-icons/react/outline'

export default function Page() {
    return <SettingsIcon strokeWidth={2} size={24} />
}
```

## SolarProvider and RSC

`SolarProvider` uses React Context and CSS variables. Use it in a client component.

```tsx
'use client'
import { SolarProvider } from '@solar-icons/react'
import { HeartIcon } from '@solar-icons/react/bold'

export function IconProvider({ children }: { children: React.ReactNode }) {
    return (
        <SolarProvider color="#3b82f6" size={24}>
            {children}
        </SolarProvider>
    )
}
```

Icons used without a provider work in server components.

## Vue SSR (Nuxt)

```vue
<template>
    <HeartIcon color="#ef4444" :size="32" />
</template>

<script setup>
import { HeartIcon } from '@solar-icons/vue/bold/heart'
</script>
```

## SvelteKit

Svelte icons work in SvelteKit SSR. Import in your server-side components.

## Dynamic icons (RSC)

Dynamic icons are also server-compatible. `DynamicIcon` is a pure component with no hooks and no `'use client'` directive.

```tsx
import { HeartIcon } from '@solar-icons/react/dynamic'

export default function Page() {
    return (
        <>
            <HeartIcon weight="Bold" color="#ef4444" />
            <HeartIcon weight="Linear" color="#3b82f6" size={32} />
        </>
    )
}
```

## No more `/ssr` subpath

The legacy `@solar-icons/react/ssr` path no longer exists. Import directly from the style path.
