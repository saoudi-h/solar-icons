v3 splits the package into two import paths. Pick the one that matches your usage.

- **Per-style path** (default): one style per component, tree-shakable at the import level. Use this for most apps.
- **Dynamic path**: a single component that bundles all six styles and switches via `weight`. Use it only when you swap styles at runtime.

## Import paths

Kebab-case style directories.

**Before (pre-v3):**

```js
import { Home } from '@solar-icons/vue/Bold'
```

**After (recommended — per-style):**

```js
import { HomeIcon } from '@solar-icons/vue/bold'
```

**After (only for runtime switching — dynamic):**

```js
import { HomeIcon } from '@solar-icons/vue/dynamic'
```

Top-level barrel exports disambiguated names:

```js
import { HomeBoldIcon } from '@solar-icons/vue'
```

## Component naming

When the style is in the import path, the name drops the style suffix and gains an `Icon` suffix (`Home` → `HomeIcon`). See the [React migration](/docs/v3/migration-to-v3/react#component-naming) for the full naming rules. Separate icon names were also renamed in v3 — see the [Icon Renames page](/docs/v3/migration-to-v3/icon-renames).

## Category imports removed

```js
// Before — imports the entire category
import { Arrows } from '@solar-icons/vue/category'
```

**After (recommended — per-style):**

```js
import { ArrowUpIcon } from '@solar-icons/vue/bold/arrow-up'
```

**After (only for runtime switching — dynamic):**

```js
import { ArrowUpIcon } from '@solar-icons/vue/dynamic/arrow-up'
```

## `weight` prop removed

The pre-v3 single package bundled six styles per component and switched via `weight`. v3 drops `weight` from the per-style path. The dynamic path keeps it.

**Before:**

```vue
<script setup>
import { Heart } from '@solar-icons/vue'
</script>

<template>
    <Heart weight="Bold" />
</template>
```

**After (recommended — per-style, no `weight`):**

```vue
<script setup>
import { HeartIcon } from '@solar-icons/vue/bold'
</script>

<template>
    <HeartIcon />
</template>
```

**After (only for runtime switching — dynamic, keeps `weight`):**

```vue
<script setup>
import { HeartIcon } from '@solar-icons/vue/dynamic'
</script>

<template>
    <HeartIcon weight="Bold" />
</template>
```

## `mirrored` prop removed

Use CSS `scaleX(-1)` instead.

## New: SolarProvider

```vue
<SolarProvider color="#3b82f6" :size="24" :stroke-width="1.5">
  <App />
</SolarProvider>
```

Vue template props use kebab-case: `stroke-width`, `secondary-color`, `secondary-opacity`.

## New: Duotone and stroke width

```vue
<HeartIcon secondary-color="#f59e0b" :secondary-opacity="0.4" />
<SettingsIcon :stroke-width="2" />
```
