The old `@solar-icons/react` was the dynamic package — all six styles per component, runtime switching via `weight`. In v3, this is accessed from the `dynamic` import path.

## Import path

Same package, different import path:

**Before:**

```jsx
import { Heart } from '@solar-icons/react'
;<Heart weight="Bold" />
```

**After:**

```jsx
import { HeartIcon } from '@solar-icons/react/dynamic'
;<HeartIcon weight="Bold" />
```

## Component naming

Add `Icon` suffix: `Home` → `HomeIcon`, `ArrowRight` → `ArrowRightIcon`.

## Default namespace export removed

The old `@solar-icons/react` exported a default namespace object pulling every icon into the bundle:

```jsx
// Before — all 7000+ icons in the bundle
import solar from '@solar-icons/react'
;<solar.ArrowUp weight="Bold" />
```

This no longer exists. Import icons individually per style.

## Category imports removed

Importing an entire category also defeated tree-shaking:

**Before:**

```jsx
import { Arrows } from '@solar-icons/react/category'
;<Arrows.ArrowUp weight="Bold" />
```

**After:**

```jsx
import { ArrowUpIcon } from '@solar-icons/react/dynamic/arrow-up'
;<ArrowUpIcon weight="Bold" />
```

## SSR subpath removed

**Before:**

```jsx
import { Home } from '@solar-icons/react/ssr'
```

**After:**

```jsx
import { HomeIcon } from '@solar-icons/react/bold'
```

## New features available

`SolarProvider`, `secondaryColor`/`secondaryOpacity`, and `strokeWidth`. See the [React package page](/docs/v3/frameworks/react).
