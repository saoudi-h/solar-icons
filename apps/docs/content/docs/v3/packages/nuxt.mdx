```shell
npx nuxi module add @solar-icons/nuxt
```

Wraps `@solar-icons/vue`. Auto-imports icon components with a configurable prefix.

```ts
// nuxt.config.ts
export default defineNuxtConfig({
    modules: ['@solar-icons/nuxt'],
    solarIcons: {
        namePrefix: 'Solar',
        autoImport: true,
    },
})
```

## Auto-import

With `autoImport: true`, every icon is available in templates. Component name pattern: `{prefix}{IconName}{Style}Icon`.

```vue
<template>
    <SolarHeartBoldIcon color="#ef4444" :size="32" />
    <SolarSettingsOutlineIcon :stroke-width="2" />
</template>
```

`SolarProvider` and `useSolar` are auto-imported regardless of the `autoImport` setting.

## Static imports

For tree-shakable imports, disable `autoImport`:

```ts
export default defineNuxtConfig({
    modules: ['@solar-icons/nuxt'],
    solarIcons: { autoImport: false },
})
```

Then import directly from `@solar-icons/vue`:

```vue
<script setup>
import { HeartIcon } from '@solar-icons/vue/bold'
</script>
```

## Module options

| Option       | Type      | Default   | Description                    |
| ------------ | --------- | --------- | ------------------------------ |
| `namePrefix` | `string`  | `'Solar'` | Prefix for auto-imported names |
| `autoImport` | `boolean` | `true`    | Auto-import all icons          |

## Props and SolarProvider

Uses `@solar-icons/vue` internally. See the [Vue package page](/docs/v3/packages/vue) for props, SolarProvider, and useSolar.

`useSolar()` must be called inside a `<SolarProvider>` tree. The provider uses scoped `provide`/`inject` — no global context.

## ESM only

`@solar-icons/nuxt` is ESM-only.
