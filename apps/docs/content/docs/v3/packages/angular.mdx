```package-install
@solar-icons/angular
```

Standalone components with signal-based inputs. PascalCase component names, kebab-case selectors in templates.

## Import patterns

```ts
// Per-style (no style suffix in name)
import { HeartIcon } from '@solar-icons/angular/bold'

// Single icon
import { HeartIcon } from '@solar-icons/angular/bold/heart'

// Top-level (style suffix in name)
import { HeartBoldIcon } from '@solar-icons/angular'
```

Add icons to your component's `imports`:

```ts
import { Component } from '@angular/core'
import { HeartIcon } from '@solar-icons/angular/bold'

@Component({
    standalone: true,
    imports: [HeartIcon],
    template: '<heart-icon color="#ef4444" [size]="32" />',
})
export class AppComponent {}
```

## Duotone

```html
<heart-icon color="#3b82f6" secondaryColor="#f59e0b" [secondaryOpacity]="0.4" [size]="48" />
```

## Stroke width

```html
<settings-icon [strokeWidth]="2" [size]="24" />
```

## SolarProvider

Angular uses `<solar-provider>`. Provider duotone props are `duotoneColor`/`duotoneOpacity`.

```ts
import { SolarProviderComponent } from '@solar-icons/angular'
import { HeartIcon } from '@solar-icons/angular/bold'

@Component({
    standalone: true,
    imports: [SolarProviderComponent, HeartIcon],
    template: `
        <solar-provider color="#3b82f6" [size]="24" [strokeWidth]="1.5">
            <heart-icon />
        </solar-provider>
    `,
})
export class AppComponent {}
```

| Input            | CSS Variable              | Default        |
| ---------------- | ------------------------- | -------------- |
| `color`          | `--solar-color`           | `currentColor` |
| `size`           | `--solar-size`            | `24px`         |
| `strokeWidth`    | `--solar-stroke-width`    | `1.5`          |
| `duotoneColor`   | `--solar-duotone-color`   | —              |
| `duotoneOpacity` | `--solar-duotone-opacity` | —              |

Note: individual icons use `secondaryColor`/`secondaryOpacity`. The provider uses `duotoneColor`/`duotoneOpacity`.

## useSolar

Returns Angular `Signal`s.

```ts
import { useSolar } from '@solar-icons/angular'

const solar = useSolar()
solar.setColor('#ef4444')
```

## Dynamic icon

```ts
import { SolarDynamicIcon, provideSolarIcons } from '@solar-icons/angular'
import { HeartIcon } from '@solar-icons/angular/bold'

@Component({
    standalone: true,
    imports: [SolarDynamicIcon],
    providers: [provideSolarIcons({ HeartIcon })],
    template: '<ng-container [solarIcon]="iconName" [size]="24" color="red" />',
})
export class AppComponent {
    iconName = 'HeartIcon'
}
```

## Icon props

| Input              | Type               | Description            |
| ------------------ | ------------------ | ---------------------- |
| `color`            | `string`           | Icon color             |
| `size`             | `string \| number` | Width and height       |
| `strokeWidth`      | `string \| number` | Stroke width           |
| `secondaryColor`   | `string`           | Duotone accent color   |
| `secondaryOpacity` | `number`           | Duotone accent opacity |
| `isolated`         | `boolean`          | Ignores provider       |
| `alt`              | `string`           | Accessibility label    |
| `ariaLabel`        | `string`           | `aria-label` attribute |

## CSS variables

Same five as [React](/docs/v3/packages/react#css-variables), applied via `[style.--solar-*]` host bindings.
