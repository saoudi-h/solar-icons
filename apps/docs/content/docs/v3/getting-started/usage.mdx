Import paths use kebab-case. The icon component name follows the import path: when the style is in the path, the name drops the style suffix.

## Import patterns

Three levels of specificity.

### All icons (top-level)

Every icon from every style, with style suffixes in the component name.

```jsx
import { HeartBoldIcon, SettingsOutlineIcon } from '@solar-icons/react'
```

The component name includes the style suffix because the import path does not specify one.

### Per-style

All icons from one style. The component name drops the style suffix.

```jsx
import { HeartIcon, BellIcon } from '@solar-icons/react/bold'
```

The style is in the path (`/bold`), so the name is `HeartIcon`.

### Single icon (tree-shakable)

One icon from one style. Your bundler removes everything else.

```jsx
import { HeartIcon } from '@solar-icons/react/bold/heart'
import { HeartIcon } from '@solar-icons/vue/bold/heart'
import { HeartIcon } from '@solar-icons/solid/bold/heart'
```

```svelte
import HeartIcon from '@solar-icons/svelte/bold/heart'
```

Svelte uses a default export for single-icon imports.

## Styles

Six styles, each a separate import path segment.

| Directory      | Style       | Description                  |
| -------------- | ----------- | ---------------------------- |
| `bold`         | Bold        | Solid fill                   |
| `bold-duotone` | BoldDuotone | Two-tone filled with accent  |
| `broken`       | Broken      | Outline with broken segments |
| `line-duotone` | LineDuotone | Two-tone outline with accent |
| `linear`       | Linear      | Outline with stroke          |
| `outline`      | Outline     | Outline, no stroke           |

## Basic usage

```jsx
import { HeartIcon } from '@solar-icons/react/bold'

function App() {
    return <HeartIcon color="#ef4444" size={32} />
}
```

```vue
<script setup>
import { HeartIcon } from '@solar-icons/vue/bold'
</script>

<template>
    <HeartIcon color="#ef4444" :size="32" />
</template>
```

```svelte
<script>
import HeartIcon from '@solar-icons/svelte/bold/heart'
</script>

<HeartIcon color="#ef4444" size={32} />
```

## Dynamic style switching

Import from the `dynamic` path for runtime style changes. These components accept a `weight` prop that switches between all six styles.

```jsx
import { HeartIcon } from '@solar-icons/react/dynamic'

function App() {
    return (
        <>
            <HeartIcon weight="Bold" />
            <HeartIcon weight="Linear" color="#3b82f6" size={32} />
        </>
    )
}
```

```vue
<script setup>
import { HeartIcon } from '@solar-icons/vue/dynamic'
</script>

<template>
    <HeartIcon weight="Bold" />
    <HeartIcon weight="Linear" color="#3b82f6" :size="32" />
</template>
```

Dynamic icons are also available per-icon for finer tree-shaking:

```jsx
import { HeartIcon } from '@solar-icons/react/dynamic/heart'
```
