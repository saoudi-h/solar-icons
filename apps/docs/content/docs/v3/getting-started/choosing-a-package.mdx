Solar Icons v3 ships one package per framework. Pick the one that matches your stack.

## React

`@solar-icons/react` covers React 18.3+. Use it with Next.js, Remix, Vite, or any React setup. Supports RSC and SSR out of the box.

## Vue

`@solar-icons/vue` covers Vue 3.0+. Works with Vite, Nuxt, and any Vue project. The Nuxt module wraps this package and adds auto-import.

## Svelte

`@solar-icons/svelte` covers Svelte 5.0+. Uses Svelte 5 runes (`$props`, `$derived`, `$bindable`).

## SolidJS

`@solar-icons/solid` covers SolidJS 1.6+. Leverages Solid's signals for reactive state.

## Angular

`@solar-icons/angular` covers Angular 17+. Uses signals, standalone components, and DI-based provider.

## React Native

`@solar-icons/react-native` covers React Native 0.72+. Requires `react-native-svg` as a peer dependency.

## Nuxt

`@solar-icons/nuxt` wraps `@solar-icons/vue` for Nuxt 3+. Adds auto-import of all icon components with a configurable name prefix.

## One package, two modes

Every package includes both classic and dynamic imports.

- **Classic imports** (`/bold/heart`): One style per import. Tree-shakable. Smallest bundle.
- **Dynamic imports** (`/dynamic/heart`): Accept a `weight` prop for runtime style switching. Larger bundle.

No separate `-perf` or `-reactive` packages. Choose the import path that matches your needs.
